## 1. Introduction
This Test Plan describes the testing approach, scope, objectives, and resources for the SauceDemo web application. The purpose of this document is to ensure that the application meets functional requirements, provides a good user experience, and is free from critical defects.

---

## 2. Objectives
- Verify that all core features of SauceDemo work as expected
- Ensure the application meets functional and usability requirements
- Identify and document defects with clear reproduction steps
- Validate end-to-end user flow from login to checkout completion
- Provide testing evidence suitable for a Junior QA / Software Tester portfolio

---

## 3. Scope of Testing

### 3.1 In Scope
- Login functionality
- Product inventory listing and sorting
- Product detail view
- Add to cart / Remove from cart functionality
- Cart management
- Checkout process (Information, Overview, Complete)
- UI elements and navigation
- Basic usability and visual behavior

### 3.2 Out of Scope
- Performance and load testing
- Security and penetration testing
- Backend and database validation
- Cross-browser compatibility testing
- Mobile responsiveness testing

## 5. Test Approach

### 5.1 Test Type
- Manual Functional Testing
- UI / Usability Testing
- Exploratory Testing

### 5.2 Test Design Techniques
- Requirement-based testing
- Positive and negative test cases
- Boundary value testing (input fields)
- End-to-end scenario testing

---

## 6. Test Environment
- Operating System: Windows 10 / Windows 11
- Browser: Google Chrome (latest version)
- Network: Stable internet connection
- Test Account: Standard user credentials provided by SauceDemo

---

## 7. Entry and Exit Criteria

### 7.1 Entry Criteria
- Application is accessible
- Test environment is ready
- Feature List and Test Plan are approved

### 7.2 Exit Criteria
- All planned test cases are executed
- Critical and high severity defects are reported
- Test summary is completed

---

## 8. Test Deliverables
- Feature List
- Test Plan
- Test Cases (Excel / Spreadsheet)
- Bug Reports
- Test Summary Report

---

## 9. Roles and Responsibilities
- QA Tester: Test planning, test execution, defect reporting
- Developer (Assumed): Defect fixing and clarification
- Stakeholder (Assumed): Requirement clarification

---

## 10. Risks and Mitigation
| Risk | Impact | Mitigation |
|------|--------|------------|
| Unclear requirements | Medium | Clarify behavior based on UI observation |
| Limited test scope | Low | Focus on core user flows |
| Test environment changes | Low | Re-validate environment before testing |

---

## 11. Defect Management
- Defects will be logged with the following information:
  - Defect ID
  - Title
  - Description
  - Steps to Reproduce
  - Expected Result
  - Actual Result
  - Severity and Priority
  - Status

---

## 12. Approval
This Test Plan is prepared to demonstrate QA testing knowledge and methodology for a Junior QA / Software Tester position.
